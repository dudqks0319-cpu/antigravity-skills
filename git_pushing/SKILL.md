# Git Pushing 📤

Git 커밋과 푸시를 자동화하는 스킬입니다.

## 워크플로우

1. **변경사항 확인**: `git status`로 변경 파일 확인
2. **요약 보고**: 한글로 변경사항 요약
3. **승인 요청**: 커밋 전 사용자 확인
4. **커밋 실행**: 승인 후 커밋
5. **푸시**: 원격 저장소에 업로드

## 커밋 메시지 규칙

```
[타입]: [설명]

타입:
- feat: 새 기능
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 스타일 변경
- refactor: 리팩토링
```

## 보안 규칙

> ⚠️ 푸시(Push) 전에 **반드시** 사용자 승인을 받는다.

## 예시 출력

```
📋 변경사항 요약:
- index.html: 헤더 수정
- style.css: 다크모드 추가

💬 커밋 메시지: "feat: 다크모드 지원 추가"

이대로 커밋(Commit)하고 푸시(Push)해도 될까요?
```
