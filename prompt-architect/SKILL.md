# Prompt Architect

이 스킬은 사용자가 "웹사이트 만들어줘", "앱 개발하고 싶어"와 같이 **모호하거나 구체적인 명세가 없는 요청**을 했을 때 자동으로 발동합니다.
개발 지식이 없는 사용자도 전문가 수준의 결과물을 얻을 수 있도록, 역으로 질문하고 제안하여 요청을 구체화(Refining)합니다.

## 발동 조건
*   사용자의 요청이 1-2문장으로 매우 짧거나 모호할 때.
*   기술 스택(React, Python 등)이나 디자인 스타일, 구체적인 기능 명세가 빠져 있을 때.

## 동작 방식 (Workflow)

### 1단계: 분석 및 옵션 제안 (Analysis & Options)
사용자의 의도를 파악하고, 가능한 3가지 정도의 구체적인 **"시나리오(Scenario)"**를 제안합니다. 각 시나리오는 서로 다른 기술 스택과 분위기를 가집니다.

**예시 답변 포맷:**
> "어떤 웹사이트를 원하시는지 조금 더 구체적으로 잡아드릴까요? 아래 3가지 옵션 중 마음에 드는 방향을 골라주세요!"
>
> **Option A: 빠르고 트렌디한 스타트업 스타일 🚀**
> *   **특징**: 화려한 애니메이션, 모바일 친화적, 어두운 배경(Dark Mode).
> *   **추천 기술**: React + Vite + TailwindCSS.
> *   **적합**: 포트폴리오, 홍보용 랜딩 페이지.
>
> **Option B: 깔끔하고 신뢰감 있는 기업 스타일 💼**
> *   **특징**: 흰색 배경, 가독성 높은 폰트, 정돈된 레이아웃.
> *   **추천 기술**: HTML/CSS (Vanilla) 또는 Next.js.
> *   **적합**: 회사 소개 페이지, 블로그.
> 
> **Option C: 데이터 중심의 대시보드 스타일 📊**
> *   **특징**: 차트, 표, 관리자 페이지 느낌.
> *   **추천 기술**: Python (Streamlit) 또는 Vue.js.
> *   **적합**: 관리 도구, 데이터 시각화.

### 2단계: 상세 사항 조율 (Fine-tuning)
사용자가 옵션을 선택하면, 추가로 필요한 디테일(색상, 필수 기능, 타겟 유저)을 2-3가지 질문으로 좁힙니다.

### 3단계: 최종 프롬프트 생성 (Final Prompt Generation)
수집된 정보를 바탕으로 **"개발자에게 전달할 완벽한 프롬프트"**를 대신 작성해서 보여줍니다.

> **작성된 최종 프롬프트:**
> "React와 TailwindCSS를 사용하여 모던한 다크 모드 포트폴리오 웹사이트를 만들어줘. 메인 섹션에는 타이핑 애니메이션이 들어간 자기소개가 필요하고, 스크롤 할 때 페이드인 효과가 있는 프로젝트 갤러리 섹션을 구현해줘."

이후 사용자가 "좋아, 이걸로 진행해"라고 하면 실제 개발을 시작합니다.
